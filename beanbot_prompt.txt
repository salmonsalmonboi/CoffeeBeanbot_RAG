# Role
You are a Senior AI & Python Engineer specializing in Local RAG systems, Edge AI, and Document Parsing. Your coding style is modular, well-documented, and production-ready with robust error handling.

# Task
Build the "Coffee Bean Sorter Assistant", a RAG-based Chatbot designed for Knowledge Handover. This system will help junior engineering students query a Thai thesis and an English research paper regarding an automated coffee bean sorting system (incorporating YOLOv12, computer vision, and Arduino-based hardware for real-time defect ejection).

# Architecture & Tech Stack
- Language: Python 3.10+
- Ingestion/Parsing: `docling` (Crucial for handling complex Thai PDFs, tables, and academic formats)
- Chunking & Orchestration: LangChain
- Embeddings: HuggingFace (`all-MiniLM-L6-v2` running locally)
- Vector Database: ChromaDB (Local persistent storage)
- LLM: Gemini 1.5 Flash (via Google GenAI SDK or LangChain Google integration) or You can Suggest me for newbie
- UI: Streamlit

# Domain Context
The knowledge base contains deep technical details on:
- Data annotation and dataset preparation for coffee beans.
- Ultralytics YOLO model training, optimization, and parameter tuning.
- Real-time deployment constraints on edge devices.
- Hardware integration: Conveyor belt synchronization, camera setup, and Arduino-controlled ejection mechanisms.

# Project Structure Requirements
Please build the project step-by-step into three modular scripts:

1. `01_ingest_knowledge.py`:
   - Read PDF files from `data/raw_docs/`.
   - Use `docling` to extract text and convert to Markdown.
   - Use a text splitter (e.g., `RecursiveCharacterTextSplitter`) with a chunk size of ~1200 and overlap of ~200 to maintain context.
   - Embed and store chunks in a local ChromaDB at `data/chroma_db/`.

   - Inside data folder you wil see my file.
   1.1. `.pdf`: Continue using `docling` (or PyPDFLoader).
   1.2. `.md` (e.g., README.md): Use `UnstructuredMarkdownLoader` or basic TextLoader.
   1.3. `.py` (e.g., Baseline_Capture.py): Use `LanguageParser` or `RecursiveCharacterTextSplitter.from_language` with `Language.PYTHON` to preserve code structure.
   1.4. `.yaml` / `.yml` (e.g., args.yaml, data.yaml): Read as text using `TextLoader` so the LLM can see the exact configuration keys and values.

Make sure the script loops through the folder, identifies the file extension, applies the correct loader, and then embeds all resulting chunks into the same ChromaDB instance.

2. `02_rag_pipeline.py`:
   - Implement the retrieval and generation logic.
